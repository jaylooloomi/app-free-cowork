# Free Claude Code — 人工驗收清單(約 5-10 分鐘)

開發機自動化測試已全數通過(61 項單元測試、本機端對端實測:真實雲端模型成功將 10 個檔案分類)。
以下項目需要**真人與真帳號**才能驗證,請逐項勾選。

## A. 基本流程(本機,環境已就緒)

- [ ] 雙擊 `launcher\src-tauri\target\release\bundle\nsis\Free Claude Code_0.1.0_x64-setup.exe` 安裝(應無 UAC 提示)
- [ ] 安裝後按 `Alt+H` → 輸入面板彈出、游標已聚焦
- [ ] 狀態列顯示「就緒 · minimax-m2.5:cloud」(或設定的模型)
- [ ] 輸入「幫我整理桌面,並且建立資料夾分類」→ Enter → 跳出終端機視窗、Claude 開始工作
  - ⚠️ 建議先用測試資料夾:設定 → 工作目錄 指到一個拋棄式資料夾再測
- [ ] 終端機裡的互動介面正常(可打字續聊)— 這項只能在**安裝版**驗證,dev 模式行為不同
- [ ] `Esc` 與點擊面板外面會關閉面板;`↑` 能叫回歷史輸入

## B. 設定頁

- [ ] 系統匣圖示 → 設定:六個欄位都能改、儲存後顯示「已儲存 ✓」
- [ ] 把快捷鍵改成 `Ctrl+Alt+Space` → 儲存 → 新鍵立即生效、`Alt+H` 失效;改回來
- [ ] 故意輸入無效快捷鍵(如 `XYZ`)→ 儲存顯示錯誤、**原快捷鍵仍有效**(回滾機制)
- [ ] 勾「背景模式」→ 送出一個小任務 → 不開終端機,完成後跳 Windows 通知
- [ ] 設定 → 開啟記錄資料夾:能看到 `fcc-*.log` 執行紀錄

## C. 首次使用流程(需要一台沒裝過的機器或 Windows Sandbox)

> 開發機已裝好所有元件,此流程無法在開發機重現。
> 啟用 Windows Sandbox 後雙擊 `scripts\e2e-sandbox.wsb` 可半自動驗證(報告在 `scripts\sandbox-out\report.txt`),
> 或在乾淨 VM 手動執行下列步驟:

- [ ] 全新機器跑 `install.ps1` 一行安裝 → 裝完 app 自動啟動
- [ ] `Alt+H` → 輸入任務 → 精靈出現,依序安裝 Ollama(約 1.3GB,介面有標註)與 Claude Code
- [ ] 「登入 ollama.com」按鈕 → 瀏覽器開啟 → 完成登入(**需要你的 ollama.com 帳號**)→ 精靈自動繼續
- [ ] 聲明頁 → 「開始使用」→ 剛剛輸入的任務**自動執行**(不用重打)
- [ ] 全程沒有出現任何 UAC 管理員提示

## D. 異常情境(可選)

- [ ] 拔網路 → `Alt+H` → 狀態列顯示「離線 — 雲端模型需要網路連線」、無法送出
- [ ] 雲端額度用完時(若遇到)→ 通知顯示「免費額度已用完,稍後重置」

## E. 解除安裝

- [ ] `uninstall.ps1`(或設定 → Windows 應用程式解除安裝)→ app 移除、`%APPDATA%\free-claude-code` 清除

- 背景模式的完成通知不可點擊開啟記錄(請從設定開記錄資料夾)
- Windows Sandbox 全新機器流程未在開發機實跑(功能未啟用,需管理員+重開機);工件已備妥
